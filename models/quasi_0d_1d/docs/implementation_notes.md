# Quasi 0-D/1-D Implementation Notes

## Status

Accepted go-to quasi 0-D/1-D model. This document is the engineering contract
for the PhysioBlocks-only quasi vessel-chain topology, accepted calibration
policy, waveform policy, and validation artifacts.

## Scope and Canonical Configs

The quasi model is generated from the accepted full 0-D configs by replacing
selected aortic and Fontan shortcut paths with distributed R-L-C chains while
retaining the accepted heart, active atrium, systemic beds, pulmonary beds, and
fenestration.

Canonical configs:

```text
models/quasi_0d_1d/configs/fontan_quasi_smoke.jsonc
models/quasi_0d_1d/configs/fontan_quasi_baseline.jsonc
models/quasi_0d_1d/configs/fontan_quasi_vasodilation.jsonc
models/quasi_0d_1d/configs/fontan_quasi_fenestration.jsonc
models/quasi_0d_1d/configs/fontan_quasi_lpa_obstruction.jsonc
```

## Topology and Naming

Quasi-chain block names use the `quasi_<vessel>_<element>_<index>` pattern.
The canonical vessels are `aao_arch`, `dao`, `svc`, `ivc`, `rpa`, and `lpa`.
The old full 0-D conduit pressure nodes are removed from accepted quasi
configs.

## Block and Numerical Conventions

Each quasi segment uses one `hydraulic_rl_block` and one downstream `c_block`.
Positive flow follows node `1 -> 2`. The model remains PhysioBlocks-only and
does not solve nonlinear 1-D PDEs.

## Parameter and Unit Conventions

Quasi chain resistance, inertance, and compliance are SI-valued hydraulic
parameters. Pressures are stored in pascal, flows in cubic metres per second,
volumes in cubic metres, and metrics report the corresponding clinical units.
Dimensionless calibration factors are recorded separately from generated chain
priors.

## Calibration and Validation Policy

Baseline is the only calibration case. Scenario configs inherit the accepted
baseline and are validation cases. Superiority requires the frozen gate:
stability, hard clinical score, aggregate direct score, paper comparison, pump
and Fontan/pulmonary non-regression, waveform no-regression, and at least one
quasi-specific vascular improvement.

## Scenario Policy

Vasodilation, fenestration, and LPA obstruction are generated by parameter
changes only. The LPA obstruction scenario doubles the LPA quasi-chain
resistance without scenario-specific retuning.

## Documentation Regeneration

When quasi topology, chain counts, parameterization, behavior, or accepted
reference outputs change, update this file, `README.md`, the SVG schematic, PNG
schematic export, and the generated technical reference together.

```bash
python3 scripts/modeling/build_quasi_configs.py --check
python3 scripts/docs/build_model_reference_pdfs.py --model quasi_0d_1d
python3 scripts/docs/check_model_docs.py --model quasi_0d_1d
```

## Current Limitations

The model is quasi 0-D/1-D only. Its chains are lumped R-L-C approximations,
not true nonlinear 1-D vessels. The model is not clinically validated.

## Detailed Engineering Notes

### Scope

The quasi 0-D/1-D model is a PhysioBlocks-only closed-loop Fontan model. It is
generated from the accepted full 0-D scenario configs by removing selected
0-D conduit shortcuts and inserting accepted R-L-C chain fragments.

The executable configs are:

```text
models/quasi_0d_1d/configs/fontan_quasi_smoke.jsonc
models/quasi_0d_1d/configs/fontan_quasi_baseline.jsonc
models/quasi_0d_1d/configs/fontan_quasi_vasodilation.jsonc
models/quasi_0d_1d/configs/fontan_quasi_fenestration.jsonc
models/quasi_0d_1d/configs/fontan_quasi_lpa_obstruction.jsonc
```

They are generated by:

```bash
python3 scripts/modeling/build_quasi_configs.py
```

### Topology

The retained closed-loop topology is:

```text
active atrium E(t)
  -> AV valve R-L -> active spherical ventricle
  -> aortic valve R-L -> aortic quasi chains
  -> systemic beds -> caval quasi chains
  -> TCPC node -> pulmonary quasi chains
  -> pulmonary RCR beds -> active atrium E(t)
```

The quasi replacements are:

| Chain | Implemented path | Segments |
|---|---|---:|
| AAo/arch x4 | `aao -> aortic_arch` | 4 |
| DAo x6 | `aortic_arch -> dao` | 6 |
| SVC x3 | `svc -> tcpc` | 3 |
| IVC x5 | `ivc -> tcpc` | 5 |
| RPA x3 | `tcpc -> rpa` | 3 |
| LPA x4 | `tcpc -> lpa` | 4 |

Each R-L-C chain cell is represented as:

```text
hydraulic_rl_block: upstream pressure node -> downstream pressure node
c_block: downstream pressure node compliance
```

The positive flow orientation follows node `1 -> 2`.

### Parameter Sources

`scripts/modeling/derive_quasi_vessel_parameters.py` derives first-pass vessel
priors from:

```text
data/processed/aramburu_2024/model_inputs/aorta_geometry.csv
data/processed/aramburu_2024/model_inputs/fontan_cross_geometry.csv
data/processed/aramburu_2024/targets/target_policy.csv
models/full_0d/configs/fontan_0d_baseline.jsonc
```

The accepted executable configs use:

```text
models/quasi_0d_1d/config_fragments/quasi_vessel_chains_corrected.json
models/quasi_0d_1d/calibration/calibration_factors.json
```

The accepted aortic chains use corrected trunk resistance with retained
geometry-derived inertance and chain compliance. SVC, IVC, RPA, and LPA chain
resistances start from calibrated full 0-D pathway priors.

### Calibration Policy

Baseline is the calibration case. Vasodilation, fenestration, and LPA
obstruction are validation cases and are not retuned.

The accepted baseline uses bounded, interpretable scales:

```text
heart_contractility_scale = 1.05
upper_systemic_resistance_scale = 1.00
lower_systemic_resistance_scale = 1.12
right_pulmonary_total_resistance_scale = 1.15
left_pulmonary_total_resistance_scale = 1.15
right_pulmonary_proximal_fraction = 0.65
left_pulmonary_proximal_fraction = 0.65
endpoint aortic compliance scale = 0.02
terminal lower arterial compliance scale = 0.5
lower systemic proximal resistance fraction = 0.95
upper/lower venous compliance scale = 0.95
SVC/IVC compliance scale = 0.95
active atrium unstressed-volume scale = 1.05
heart radius/thickness scale = 1.01
```

All quasi chain R/L/C scale factors remain `1.00` in the accepted baseline
except for scenario-specific LPA obstruction resistance scaling.

### Metrics

`scripts/metrics.py` is model-family aware. For quasi chains, it treats the
first R-L segment flux as vessel inlet flow and the final R-L segment flux as
vessel outlet flow.

Standardized cycle metrics are:

```text
mean_<vessel>_inlet_flow_ml_s
mean_<vessel>_outlet_flow_ml_s
integral_<vessel>_inlet_flow_ml
integral_<vessel>_outlet_flow_ml
<vessel>_cycle_storage_ml
<vessel>_mass_balance_rel
```

The signed storage value is inlet integral minus outlet integral over the last
cardiac cycle. The relative mass-balance value normalizes absolute storage by
the sum of absolute inlet and outlet cycle integrals.

### Aortic Waveform Policy

`calibration/aortic_signal_policy.json` defines the accepted aortic waveform
mapping:

```text
Q_AAo -> valve_arterial.flux
Q_DAo clinical -> lower_ra4.flow
Q_DAo chain health -> quasi_dao_rl_06.flux
```

Clinical DAo bed-entry flow is reported as a soft diagnostic because it mixes
aortic trunk behavior with downstream systemic-bed/terminal-load behavior. DAo
chain-health flow remains the accepted aortic trunk waveform gate.

### Acceptance Artifacts

The accepted comparison to the full 0-D reference is stored in:

```text
calibration/current_quasi_gate_status.json
calibration/current_quasi_gate_status.md
calibration/quasi_superiority_gate.json
calibration/full0d_reference_scores.json
calibration/aortic_profile.json
```

The current accepted model passes stability, hard clinical score, aggregate
direct score, paper-model score, pump non-regression, Fontan/pulmonary
non-regression, AAo/DAo chain-health waveform no-regression, and the
quasi-specific vascular-improvement criterion.

### Schematic Convention

`docs/quasi_0d_1d_schematic.svg` follows the full 0-D schematic style and
component set. The quasi-specific labels mark the accepted R-L-C chain counts
and the narrowed LPA pressure node. Update the SVG first and regenerate
`docs/quasi_0d_1d_schematic.png` whenever topology or chain counts change.

`docs/quasi_0d_1d_technical_reference.md` and the generated
`docs/quasi_0d_1d_technical_reference.pdf` provide the standardized long-form
model definition with equations, segment inventory, and free-parameter list.
Regenerate them with `python3 scripts/docs/build_model_reference_pdfs.py --model
quasi_0d_1d` whenever topology or parameter values change.

### Current Limitations

The model is a computational development and calibration artifact. It is not
clinically validated. True nonlinear 1-D aorta/TCPC coupling is outside this
model family and belongs in the coupled 0-D/1-D model work.
