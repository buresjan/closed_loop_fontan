# Implementation Notes

## Current state

Task 006 makes the quasi 0-D/1-D family executable. The runnable configs live in:

```text
models/quasi_0d_1d/configs/fontan_quasi_smoke.jsonc
models/quasi_0d_1d/configs/fontan_quasi_baseline.jsonc
models/quasi_0d_1d/configs/fontan_quasi_vasodilation.jsonc
models/quasi_0d_1d/configs/fontan_quasi_fenestration.jsonc
models/quasi_0d_1d/configs/fontan_quasi_lpa_obstruction.jsonc
```

They are generated by:

```bash
.venv/bin/python scripts/modeling/build_quasi_configs.py
```

The generator starts from the calibrated full 0-D scenario configs, removes the
full 0-D vessel/conduit shortcuts, and inserts the tracked R-L-C chain fragment:

```text
models/quasi_0d_1d/config_fragments/quasi_vessel_chains.json
```

## Implemented topology

The quasi model keeps the full 0-D pump, valve, systemic bed, pulmonary bed,
and fenestration components:

```text
active atrium E(t)
  -> AV valve R-L -> active spherical ventricle
  -> aortic valve R-L -> aortic quasi chains
  -> systemic beds -> caval quasi chains
  -> TCPC node -> pulmonary quasi chains
  -> pulmonary RCR beds -> active atrium E(t)
```

The first executable release replaces these full 0-D pathways with repeated
`hydraulic_rl_block` plus `c_block` cells:

| Chain | Implemented path | Segments |
|---|---|---:|
| AAo/arch x4 | `aao -> aortic_arch` | 4 |
| DAo x6 | `aortic_arch -> dao` | 6 |
| SVC x3 | `svc -> tcpc` | 3 |
| IVC x5 | `ivc -> tcpc` | 5 |
| RPA x3 | `tcpc -> rpa` | 3 |
| LPA x4 | `tcpc -> lpa` | 4 |

The BCA, LCCA, and LSA upper-body branches remain 0-D resistive branches for
the first quasi release. The old conduit nodes (`svc_conduit`, `ivc_conduit`,
`rpa_conduit`, `lpa_conduit`) are removed. `valve_rl_block` remains only in the
atrioventricular and aortic valves.

## Chain parameter derivation

`scripts/modeling/derive_quasi_vessel_parameters.py` reads:

```text
data/processed/aramburu_2024/model_inputs/aorta_geometry.csv
data/processed/aramburu_2024/model_inputs/fontan_cross_geometry.csv
data/processed/aramburu_2024/targets/target_policy.csv
models/full_0d/configs/fontan_0d_baseline.jsonc
```

For each geometry cell it computes:

```text
Abar = 0.5 * (pi * r_in^2 + pi * r_out^2)
Lhyd = rho * length / Abar
Ctot = Abar * length / (rho * c^2)
Rgeom = tapered Poiseuille resistance
```

with:

```text
rho = 1060 kg/m^3
mu = 0.0035 Pa*s
aortic wave speed prior = 5.35 m/s
Fontan/TCPC wave speed prior = 2.81 m/s
```

## Resistance policy

The aortic quasi chains use geometry/friction resistance. They do not preserve
the excessive full 0-D AAo-to-DAo pressure drop. Later quasi calibration should
keep most systemic pressure loss in the systemic beds unless data justify a
larger aortic trunk loss.

The SVC, IVC, RPA, and LPA quasi chains start from the calibrated full 0-D
pathway resistances. The current full 0-D Fontan pressures and RPA/LPA flow
split are accepted baseline physiology, so these values are useful priors.

Direct DAo pressure remains diagnostic/low-weight. IVC flow remains
mass-closure dependent and should be compared against raw direct,
implied-from-CO, and implied-from-pulmonary-closure targets.

## Scenario assembly

The smoke, baseline, vasodilation, fenestration, and LPA obstruction quasi
configs inherit the matching full 0-D scenario settings before shortcut
replacement.

Vasodilation and fenestration keep their scenario changes in the retained
pulmonary bed and fenestration parameters. The LPA obstruction scenario also
sets:

```text
quasi_lpa.narrowing_resistance_scale = 2.0
```

and doubles the total LPA quasi-chain resistance while leaving the chain
inertance and compliance unchanged for this first release.

## Metrics convention

`scripts/metrics.py` is model-family-aware. For quasi chains, it treats the
first R-L segment flux as the vessel inlet flow and the final R-L segment flux
as the vessel outlet flow. Segment-level flux outputs are retained for every
quasi R-L block, so internal waves/storage can be inspected directly.

For each quasi chain, the standardized cycle metrics are:

```text
mean_<vessel>_inlet_flow_ml_s
mean_<vessel>_outlet_flow_ml_s
integral_<vessel>_inlet_flow_ml
integral_<vessel>_outlet_flow_ml
<vessel>_cycle_storage_ml
<vessel>_mass_balance_rel
```

The signed storage value is inlet integral minus outlet integral over the last
cardiac cycle. The relative mass-balance value normalizes the absolute storage
by the sum of absolute inlet and outlet cycle integrals.

The tracked Task 007 reference outputs live under:

```text
models/quasi_0d_1d/reference_outputs/
```

`scenario_comparison.txt` is generated by `scripts/compare_scenarios.py` from
the four baseline/intervention metrics JSON files.

## Config fragment convention

Each R-L-C chain cell is represented by:

```text
hydraulic_rl_block: source pressure node -> downstream pressure node
c_block: downstream pressure node compliance
```

The orientation follows the local hydraulic block convention:

```text
node 1 -> node 2 is positive physical flow
```

The config builder interpolates initial pressure states for the new internal
chain nodes from the retained upstream/downstream pressure states in the source
full 0-D scenario.

## Schematic convention

`docs/schematic.svg` follows the full 0-D schematic style and component set.
The quasi-specific labels mark the implemented R-L-C chain counts and the
narrowed LPA pressure node. Update both the SVG and PNG whenever the quasi
