# Open-Loop 1-D Submodel Validation

Task: 011 - Build 1-D open-loop submodels

Status: reference validation specs implemented. No coupled closed-loop model is
accepted yet.

## Scope

Task 011 creates reproducible open-loop reference specifications for:

- `models/coupled_0d_1d/configs/submodel_aorta_1d_openloop.jsonc`;
- `models/coupled_0d_1d/configs/submodel_tcpc_1d_openloop.jsonc`;
- `models/coupled_0d_1d/configs/submodel_aorta_tcpc_1d_openloop.jsonc`.

These specs are JSON files under `models/coupled_0d_1d/configs/`. They are not
PhysioBlocks forward-simulation launcher configs. They are model-local
reference specs that bind the tracked Aramburu/Nektar geometry, measured
inflows, reference 1-D outputs, clinical comparison targets, and validation
tolerances before Task 012 attempts closed-loop coupling.

The specs are generated by:

```bash
python3 scripts/modeling/derive_1d_geometry.py
```

and validated by:

```bash
python3 scripts/calibration/validate_1d_submodels.py
```

The generated geometry summary is:

```text
models/coupled_0d_1d/calibration/one_d_openloop_geometry.json
```

## Topology

the source geometry:

| Segment | Domain | Nodes |
|---|---:|---|

| Thoracic aorta | 2 | `arch_junction` to `dao_outlet` |

| Carotic left | 4 | `arch_junction` to `left_carotid_outlet` |

The TCPC spec contains five patient-specific source segments:

|---|---:|---|

The combined spec maps aorta domains 1-4 and TCPC domains 5-9 in the tracked
combined Nektar output.

## Segment Parameters

Each source segment is represented as a generated fixed-size true 1-D segment
compatible with the Task 010 kernel. For each segment:

$$A_0 = \frac{\pi r_{\mathrm{in}}^2 + \pi r_{\mathrm{out}}^2}{2}.$$

The reference wall stiffness is selected from the target wave speed:

$$\beta = \frac{2\rho c_0^2}{\sqrt{A_0}}.$$

The generator also stores geometry-derived resistance, inertance, compliance,
and three equal subcells for the fixed 3-cell prototype block.

## Validation Gates

The validator checks:

- all referenced input, coupling, and Nektar files exist;
- all geometry is positive and each Nektar domain length matches the source
  segment length;
- selected Nektar domain samples contain pressure, flow, area, time, and
  distance columns with positive area;
- pressure/flow reference waveforms match clinical comparison targets within
  documented mean and RMSE tolerances;
- RPA/LPA flow fraction is within 0.08;
- TCPC and combined inflow/outflow mass balance is within 5 ml/s;
- reference outlet flow signs are stable.

DAo flow uses a broad waveform RMSE tolerance while retaining a tighter mean
flow tolerance. This follows the accepted aortic signal policy: downstream DAo
or bed-entry flow is sensitive to terminal-load dynamics and should not be used
to hide hard pressure or mass-balance failures.

## Limitations

- The specs validate reference open-loop geometry and Nektar/clinical targets;
  they do not yet run a generated multi-segment local PhysioBlocks network.
- The Task 010 block is fixed to three cells. Task 012 or a later generator
  must decide whether to emit many fixed-size segment blocks or introduce
  generated classes for larger segment resolutions.
- Boundary closures are not finalized. Characteristic, RCR, and measured-flow
  boundary handling remain future implementation work.
- Area positivity is still diagnostic in the local numerical kernel.
